# 人才打标系统 PRD 初稿

## 1. 文档信息
- 产品名称：人才打标系统
- 文档版本：V0.1
- 文档日期：2026-03-31
- 文档状态：初稿

## 2. 项目背景
企业需要基于员工基础信息形成标准化标签，用于人才盘点、群体筛选、统计分析和后续人才运营。当前标签定义和打标口径通常依赖人工维护，存在口径不统一、规则分散、范围不清晰、结果不可追溯等问题。

因此需要建设一套统一的人才打标系统，实现标签、规则、范围和结果的集中管理，保证规则配置标准化、执行自动化、结果可解释。

## 3. 产品目标
- 统一定义标签与标签类目，避免口径分散。
- 通过角色权限控制标签和规则的维护行为。
- 基于人员基础信息自动完成规则打标。
- 支持规则适用范围控制，包括组织、岗位序列、职位族。
- 支持互斥类目和可并存类目。
- 支持查看某个员工命中标签的具体原因。
- 支持批量重算，满足规则调整后的统一刷新。
- 支持模拟打标，用于规则发布前验证。

## 4. 适用范围
本期系统面向内部人才管理场景，覆盖：
- 人员基础属性规则打标
- 统一标签资产管理
- 统一规则配置与发布
- 模拟打标验证
- 标签结果查询与解释

本期不包含：
- 机器学习标签推荐
- 复杂统计看板
- 实时事件流计算
- 复杂审批流引擎

## 6. 核心业务概念

### 6.1 标签类目
标签类目用于对标签进行分组。

示例：
- 技能
- 性格
- 专业
- 职级
- 院校层级
- 人才属性

### 6.2 标签
标签是最终输出结果，例如：
- P7
- 20-30年龄段
- 高级工程师
- 985
- 211

### 6.3 打标规则
规则定义“什么条件命中什么标签”。

示例：
- 职级为 P7，输出标签 P7
- 年龄在 20 到 30 岁之间，输出标签 20-30年龄段
- 职称为高级工程师，输出标签 高级工程师

### 6.4 规则适用范围
范围用于限定哪些员工会参与这条规则判断。

本期支持维度：
- 组织
- 岗位序列
- 职位族

示例：
某条“高级工程师”规则仅适用于：
- 组织属于产研中心及下级
- 岗位序列为研发

### 6.5 智能标签任务
智能标签任务用于通过大模型进行语义打标，不适合用结构化规则表达的场景可通过任务配置完成。

示例：
- 根据毕业院校识别 985 / 211 / 双一流
- 根据所学专业识别专业方向标签

### 6.6 打标结果
打标结果是员工最终拥有的标签集合，并保留命中来源和解释信息。

## 7. 业务规则

### 7.1 标签类目互斥规则
- 互斥类目下，一个员工同一时间只能有一个标签。
- 可并存类目下，一个员工可同时拥有多个标签。

示例：
- 年龄段类目下，27 岁员工只能拥有“20-30年龄段”。
- 人才属性类目下，员工可以同时拥有“高级工程师”“产研人员”“核心岗位”。

### 7.2 范围生效规则
规则必须先满足适用范围，再判断命中条件。

规则执行顺序：
1. 判断规则是否已发布且在生效期内。
2. 判断员工是否在规则适用范围内。
3. 判断员工属性是否满足命中条件。
4. 若命中，则进入结果归并流程。

### 7.3 冲突处理规则
对于互斥类目，如同一员工命中多个标签，系统按以下顺序保留：
1. 条件更精确的规则优先。
2. 若仍冲突，则按系统预设冲突处理策略保留其一，并记录冲突日志。

### 7.4 年龄计算规则
年龄不作为固定字段维护，系统根据出生日期实时计算。

### 7.5 规则状态规则
规则状态包括：
- 草稿
- 已发布
- 停用

规则状态约束：
- 新建规则或编辑规则后，保存即进入草稿状态。
- 草稿规则可以直接发布。
- 草稿规则若未经过模拟打标即发布，系统需弹出提醒。
- 已发布规则参与正式打标。
- 停用规则不参与正式打标。
- 草稿、已发布、停用规则均可进行模拟打标。

## 8. 功能需求

### 8.1 标签类目管理
功能说明：以列表页方式统一维护标签类目，支持搜索、批量操作、详情查看和弹窗编辑。

功能点：
- 按类目名称搜索
- 新建类目
- 编辑类目
- 查看类目详情
- 批量启用
- 批量停用
- 批量导出

页面结构：
- 轻量搜索区：单输入框 + 查询/重置
- 批量操作区：基于勾选结果执行启用、停用、导出、新建
- 列表区：展示类目核心字段和单条编辑入口
- 分页区：展示总数、当前区间、页码切换
- 详情弹窗：点击类目名称后查看类目详情
- 编辑弹窗：点击编辑后维护类目名称、编码、状态和说明
- 标签列表弹窗：点击“标签数量”数字后，查看该类目下有哪些标签

关键字段：
- 类目名称
- 类目编码
- 描述
- 标签数量
- 状态

说明：
- 类目定义为业务类目，例如技能、性格、专业等
- 类目编码需做唯一性校验，同类型对象内不可重复

### 8.2 标签管理
功能说明：维护统一的标签字典，供规则标签管理和智能标签管理引用。

功能点：
- 标签列表查询
- 查看标签详情
- 编辑标签
- 启用/停用标签
- 删除标签
- 导出标签
- 查看标签被哪些规则引用
- 查看标签被哪些智能标签任务引用

页面结构：
- 轻量搜索区：按标签名称、标签编码搜索
- 批量操作区：启用、停用、删除、导出、新建
- 列表区：以表格方式展示标签清单
- 分页区：展示总数、当前区间、页码切换
- 详情弹窗：点击标签名称查看标签详情
- 编辑弹窗：点击编辑后维护标签名称、所属类目、描述

关键字段：
- 标签名称
- 标签编码
- 所属类目
- 描述
- 引用来源
- 状态

说明：
- 规则标签管理通过结构化规则引用已有标签
- 智能标签管理通过大模型任务从已有标签中选择输出标签
- 编辑时，标签编码和引用来源为只读字段
- 编辑时，所属类目以下拉方式选择，数据来源于标签类目管理
- 标签编码需做唯一性校验，同类型对象内不可重复

### 8.3 规则标签管理
功能说明：以列表页方式统一管理规则，支持轻量检索、批量操作和单条规则维护。

功能点：
- 按关键字查询规则列表
- 批量发布
- 批量停用
- 批量删除
- 批量导出
- 新建规则
- 查看单条规则详情
- 编辑单条规则
- 规则名称点击查看详情弹窗
- 编辑按钮打开弹窗编辑

查询方式：
- 页面仅提供一个关键字输入框
- 默认同时按“规则名称”和“目标标签”做匹配查询

页面结构：
- 轻量搜索区：单输入框 + 查询/重置
- 批量操作区：位于列表右上角，基于勾选结果执行发布、停用、删除、导出
- 新建入口：位于列表右上角，与批量操作通过分隔线区分
- 列表区：展示规则核心字段和单条操作
- 分页区：展示总数、当前区间、页码切换
- 详情弹窗：点击规则名称查看规则详情
- 编辑弹窗：点击编辑后在当前页弹窗维护规则

列表字段：
- 勾选框
- 规则名称
- 目标标签
- 命中条件
- 适用范围
- 状态
- 操作

操作规则：
- 列表状态包括：草稿、已发布、停用
- 单条操作仅保留：编辑
- 规则名称为可点击样式，点击后弹出规则详情
- 规则编码需做唯一性校验，同类型对象内不可重复
- 新建规则或编辑规则后，保存即进入草稿状态
- 草稿规则点击发布时，如尚未进行模拟打标，系统需弹出提醒
- 只有已发布规则参与正式打标

编辑弹窗说明：
- 基本信息区包含：规则名称、字段、规则类型
- 字段选择位于基本信息区，不在命中条件区重复配置
- 字段选择采用可搜索下拉方式，便于快速定位字段
- 命中条件区仅展示与当前规则类型对应的匹配逻辑
- 支持规则类型：单值匹配、枚举匹配、区间映射
- 单值匹配和枚举匹配配置输出标签
- 区间映射按区间逐行配置输出标签，并支持新增区间、删除区间
- 编辑弹窗中已删除：优先级、目标标签、输出标签类目、映射说明

### 8.4 智能标签管理
功能说明：用于配置基于大模型的智能打标任务，适合院校分层、专业归类等不适合硬编码规则的场景。

功能点：
- 按任务名称、语义规则搜索
- 新建智能标签任务
- 查看任务详情
- 编辑任务
- 批量启用
- 批量停用
- 批量删除
- 批量导出

页面结构：
- 轻量搜索区：单输入框 + 查询/重置
- 批量操作区：启用、停用、删除、导出、新建
- 列表区：展示任务核心字段和单条编辑入口
- 详情弹窗：点击任务名称查看输入内容、语义规则、样例数量、候选标签、状态、提示词
- 编辑弹窗：维护任务名称、输入内容、语义规则、样例配置、提示词模板

列表字段：
- 勾选框
- 任务名称
- 输入内容
- 语义规则
- 样例数量
- 候选标签
- 状态
- 操作

适用场景示例：
- 根据毕业院校识别 985 / 211 / 双一流标签
- 根据所学专业识别专业方向标签

任务配置示例：
- 任务名称：院校层级识别
- 输入内容：毕业院校
- 语义规则：根据院校名称语义判断学校层级，不依赖固定院校名单硬编码
- 候选标签：985、211、双一流、普通本科
- 样例配置：提供已知院校与标签的样例对
- 提示词：根据输入的毕业院校名称，结合提供样例理解其层级语义，仅从候选标签中选择一个最合适的标签输出

说明：
- 智能标签管理与规则标签管理并列
- 规则标签管理负责结构化规则
- 智能标签管理负责大模型语义判断任务
- 智能标签任务编码需做唯一性校验，同类型对象内不可重复

### 8.5 编码唯一性校验
功能说明：系统内所有带编码的对象在新增和编辑时，都必须进行同类型对象内的唯一性校验。

校验范围：
- 标签类目编码唯一
- 标签编码唯一
- 规则编码唯一
- 智能标签任务编码唯一

校验要求：
- 保存时进行唯一性校验
- 若编码重复，前端需明确提示不可保存
- 编辑场景下，若编码未变更，不应误判为重复

### 8.6 模拟打标
功能说明：用于在规则正式生效前或停用后，对规则进行样本验证和结果预览。

功能点：
- 查看可模拟的规则
- 对草稿、已发布、停用状态规则发起模拟打标
- 查看样本人群和模拟结果
- 查看命中人数、未命中人数和主要原因
- 支持从模拟打标页直接发布规则

规则说明：
- 草稿、已发布、停用状态规则都可以进行模拟打标
- 只有已发布规则参与正式打标
- 草稿规则可以不经过模拟打标直接发布
- 草稿规则若未模拟打标即发布，系统需弹出确认提醒

### 8.7 人员标签结果查询
功能说明：查询某员工当前标签结果。

功能点：
- 按员工编号/姓名查询
- 展示员工基本信息
- 按类目展示当前标签
- 展示标签命中时间
- 展示来源规则

展示示例：
- 职级：P7
- 年龄段：20-30年龄段
- 人才属性：高级工程师、产研人员

### 8.8 标签命中解释
功能说明：查看某员工标签形成原因。

功能点：
- 查看参与过判断的规则列表
- 查看范围是否命中
- 查看条件是否命中
- 查看是否因互斥被淘汰
- 查看最终保留标签原因

### 8.9 重算任务管理
功能说明：支持批量重算。

功能点：
- 发起全量重算
- 按规则重算
- 按组织范围重算
- 按员工重算
- 查询任务执行状态
- 查看成功/失败数量

### 8.10 权限管理
功能说明：维护角色与权限。

功能点：
- 角色列表
- 用户绑定角色
- 功能权限配置

## 9. 典型业务流程

### 9.1 创建并发布规则流程
1. 规则管理员创建规则。
2. 配置命中条件。
3. 配置适用范围。
4. 保存规则，状态变为草稿。
5. 可选执行模拟打标，验证规则效果。
6. 点击发布，规则状态变为已发布。
7. 若规则未经过模拟打标，发布时系统弹出确认提醒。
8. 已发布规则进入正式打标。

### 9.2 批量重算流程
1. 管理员选择重算范围。
2. 系统生成重算任务。
3. 任务执行规则计算。
4. 写入结果表和明细表。
5. 用户查看任务结果。

### 9.3 查询解释流程
1. 查询某员工当前标签。
2. 点击某个标签查看来源规则。
3. 展示范围命中、条件命中、冲突处理和最终决策。

## 10. 非功能需求

### 10.1 可追溯性
系统必须记录：
- 标签变更记录
- 规则变更记录
- 发布和停用记录
- 打标结果命中明细

### 10.2 可解释性
系统必须支持解释：
- 为什么命中某个标签
- 为什么没有命中某个标签
- 为什么候选标签被互斥淘汰

### 10.3 可维护性
规则应采用结构化配置，不采用自由脚本，降低维护门槛和风险。

### 10.4 性能
MVP 阶段以批量重算为主，保证：
- 任务状态可跟踪
- 失败可重试
- 结果可回溯

## 11. MVP 范围
本期必须实现：
- 标签类目管理
- 标签管理
- 规则标签管理
- 规则适用范围管理
- 模拟打标
- 规则发布/停用
- 人员标签结果查询

- 复杂嵌套逻辑表达式

- 实时流式重算
- 标签分析看板
- 智能推荐规则

## 12. 成功标准
系统上线后，至少满足以下标准：
- 标签和规则口径统一管理。
- 规则适用范围可控。
- 年龄段、职级等互斥类目计算正确。
- 人才属性类标签可并存。
- 标签结果可追溯到具体规则。
- 管理员可独立完成规则配置、模拟打标和结果重算。
