# 人才打标系统 - 应用架构图描述

> 本文档用于绘制应用架构图，涵盖 MVP 已实现能力和未来演进能力。图中建议用实线框表示 MVP，虚线框表示未来迭代。

---

## 整体分层

系统从上到下分为 4 层：接入层 → 应用层 → 服务层 → 基础设施层。

前文为各层的详细展开描述，末尾"应用架构总结"为精简版架构图。

---

## 第一层：用户与渠道层

用户角色：

- 标签管理员（管理类目和标签资产）
- 规则管理员（编写和调试规则）
- 规则发布人（发布和停用规则）
- 结果查看人（查询结果和证据）
- 系统管理员（权限和全局治理）
- 业务用户 / HRBP（通过 IM 对话直接使用标签能力）

接入渠道：

- Web 管理端（当前原型，8 个页面模块，面向标签治理人员）
- IM 对话渠道（飞书 / 钉钉 / 云之家 / 企业微信）
  - 通过 IM Bot 接入，用户在对话中直接调用标签 Agent 和 Skill
  - 典型场景：HRBP 在飞书中输入"帮我找出研发中心所有核心骨干"，Agent 调用圈选 Skill 返回结果
  - 典型场景：管理员在钉钉中输入"985标签覆盖率是多少"，Agent 调用看板 Skill 返回数据
  - 典型场景：规则管理员在 IM 中输入"创建一条规则：职级>=P8 且绩效=A，打管理后备标签"，Agent 调用规则生成 Skill 生成 DSL 草稿
- Open API（RESTful，供下游系统和第三方集成调用）
- Webhook / 消息订阅（标签变更事件推送到 IM 或下游系统）

## 第二层：应用服务层

按导航组划分为 4 个应用模块 + 2 个未来模块：

### 模块一：标签管理

- 标签类目管理（新建 / 编辑 / 启用 / 失效 / 删除）
- 标签定义管理（新建 / 编辑 / 启用 / 失效 / 删除）
- 标签迁移（双向迁移、类目间标签归属调整）
- 标签合并（同义标签归并治理）
- AI 标签自动发现（AI 分析全量数据，主动发现潜在有价值的标签维度）
- AI 标签冗余检测（自动识别语义重叠标签，建议合并）
- AI 标签价值评估（使用频率、下游消费次数、业务贡献度，识别僵尸标签）

### 模块二：规则与任务

- 条件打标规则（DSL 编辑 + AI 对话辅助生成）
- 智能打标规则（语义规则配置 + 候选标签提取）
- 模拟打标（方案配置 → 规则选择 → 人群选择 → 预演对比 → 证据下钻）
- 正式打标（含重算；规则选择 → 人群选择 → 执行 → 结果对比 → 提交入库）
- 自动化打标流水线（高置信度场景 AI 自动执行 + 事后抽检，分级审批机制）
- 变更影响分析（规则修改 / 停用 / 标签合并的影响面预估和 diff 视图）
- AI 规则自动调优（根据历史打标结果和人工纠错反馈，自动建议规则阈值调整）

### 模块三：规划与方案

- 规划智能体（业务场景问卷 → AI 生成标签方案 → Excel 导出）
- 3 个预设场景（管理类 / 技术类 / 校园类）
- AI 标签体系演进建议（基于业务变化主动建议标签体系调整方向）

### 模块四：结果与权限

- 权限管理（角色定义 / 权限矩阵 / 用户绑定）
- 标签结果查询（当前通过任务结果弹窗实现）
- 标签证据查看（当前通过证据弹窗实现）
- 独立标签结果查询页
- 操作日志与审计日志页
- AI 可解释性增强（决策过程可视化 / 反事实解释 / 置信度分布 / 对比解释）

### 模块五：标签消费

- 人才圈选 / 组合筛选器
- 标签分布看板（组织 / 职级 / 序列维度的覆盖率和热力图）
- 员工标签画像（单人标签全景视图 + 标签变更时间线）
- 下游场景出口（继任计划 / 人才盘点九宫格 / 学习推荐）
- 标签使用效果反馈（下游系统使用标签后的效果回流，如管理后备实际晋升率）

### 模块六：运营监控

- 标签时效性管理（TTL / 过期预警 / 自动失效）
- 定时重算调度（cron 触发）
- 数据质量检测（字段缺失率 / 异常值检测）
- 数据同步状态监控与异常告警
- AI 标签健康度监控（覆盖率异常 / 分布偏移 / 过期标签占比，主动告警）
- AI 规则失效预警（数据分布变化导致命中率异常时主动提醒）

### 全局 AI 助手

- 统一标签助手贯穿所有页面，理解当前上下文，主动提供建议
- 意图驱动操作（"帮我把核心骨干规则从 P7 调到 P8" → 自动定位规则 → 生成修改方案 → 预估影响 → 等待确认后执行）
- 自然语言即界面（IM 渠道作为一等公民，查标签、圈人、看覆盖率在对话里完成）

## 第三层：Agent & Skill 开放层

系统将核心能力封装为可编排的 Skill，由 Agent 统一调度，支持 Web、IM、API 等多渠道接入。Agent 负责意图识别和任务编排，Skill 负责原子能力执行。

### 标签 Agent（对话入口）

- 意图识别：从用户自然语言中识别操作意图（查询 / 圈选 / 规则生成 / 方案规划 / 数据统计）
- 任务编排：根据意图组合调用一个或多个 Skill，串联多步操作
- 上下文管理：维护多轮对话状态，支持追问和修正
- 权限校验：根据用户角色过滤可用 Skill 和数据范围
- 渠道适配：统一 Agent 协议，适配 Web / 飞书 / 钉钉 / 云之家 / 企业微信等渠道的消息格式

### Skill 注册表

| Skill 名称 | 能力描述 | 调用的领域服务 | 可用渠道 |
|---|---|---|---|
| tag-query | 查询员工标签（"张三有哪些标签"） | 标签资产服务 + 执行任务服务 | Web / IM / API |
| talent-select | 人才圈选（"找出所有P7以上的核心骨干"） | 标签资产服务 + 执行任务服务 | Web / IM / API |
| tag-stats | 标签分布统计（"985标签覆盖率"） | 标签资产服务 + 执行任务服务 | Web / IM / API |
| rule-generate | 规则 DSL 生成（"创建一条规则：职级>=P8，打管理后备"） | 规则引擎服务 + 规则层 AI | Web / IM / API |
| plan-generate | 标签方案规划（"围绕管理后备设计标签体系"） | 规划层 AI | Web / IM / API |
| tag-explain | 标签证据解释（"为什么张三是核心骨干"） | 数据血缘服务 + 执行任务服务 | Web / IM / API |
| tag-compare | 标签对比（"张三和李四的标签差异"） | 标签资产服务 | Web / IM / API |
| rule-publish | 规则发布（"发布管理后备规则"） | 规则引擎服务 | Web / IM |
| tag-alert | 标签变更通知（"核心骨干标签有变更时通知我"） | 执行任务服务 + 消息推送 | IM / Webhook |
| profile-view | 员工标签画像（"展示张三的标签全景"） | 标签资产服务 + 数据血缘服务 | Web / IM |

### Skill 协议规范

- 每个 Skill 定义标准的输入 Schema（参数）和输出 Schema（结果）
- Agent 通过 Skill 注册表发现可用能力
- Skill 执行前由 Agent 做权限校验和参数校验
- Skill 执行结果统一封装为结构化响应（文本 + 卡片 + 表格 + 图表）
- IM 渠道返回消息卡片（飞书卡片 / 钉钉卡片 / 云之家卡片），Web 渠道返回富组件

### IM Bot 适配层

- 飞书 Bot：通过飞书开放平台事件订阅 + 消息卡片 API 接入
- 钉钉 Bot：通过钉钉机器人 Webhook + 互动卡片接入
- 云之家 Bot：通过云之家开放平台轻应用 + 消息接口接入
- 企业微信 Bot：通过企业微信应用消息 + 图文卡片接入
- 统一消息协议：Agent 输出标准响应，由适配层转换为各 IM 平台的消息格式

## 第四层：AI 能力层

横跨所有应用模块的 AI 能力平台：

### 规划层 AI

- 标签方案自动生成（planning-agent）
- 业务场景识别与标签推荐
- 模型 API 调用（`/api/planning-agent/analyze`）
- 本地规则降级生成
- 标签自动发现（从全量数据中挖掘高频特征，推荐新标签维度）
- 标签体系演进建议（基于业务变化和数据趋势，主动建议调整方向）

### 规则层 AI

- 自然语言 → DSL 草稿生成（AI 对话）
- 规则解释自动生成
- 对话上下文管理
- 规则自动调优（根据命中率、误判率、人工修正率，建议阈值调整）
- 规则失效预警（数据分布变化导致命中率异常时主动提醒）

### 执行层 AI

- 语义规则执行引擎（文本输入 → 候选标签匹配）
- 输入字段：毕业院校 / 任职经历 / 履历描述 / 岗位描述 / 项目经历等
- 输出：候选标签 + 语义解释摘要
- 分级自主执行（高置信度自动执行 / 中置信度待审核 / 低置信度人工确认）
- 自动化打标流水线编排

### 质量层 AI

- 置信度评分与置信度分布（候选标签概率分布，如"算法 72% / 后端 18% / 数据 10%"）
- 人工校验 / 纠错流程
- 准确率统计与持续优化反馈
- AI 标签与人工标签差异分析
- 训练数据管理

### 感知层 AI

- 数据变更感知（员工晋升 / 绩效出炉 / 组织调整 → 自动判断哪些标签需要刷新）
- 标签健康度监控（覆盖率异常 / 分布偏移 / 过期标签占比 → 主动告警）
- 数据质量感知（字段缺失率突增 → 暂停相关规则执行并通知管理员）
- 标签冗余检测（语义重叠标签自动识别，建议合并）
- 标签价值评估（使用频率 / 下游消费次数 / 业务贡献度 → 识别僵尸标签）

### 学习层 AI

- 人工纠错 → 模型优化闭环（HR 修正标签后，纠错信号回流到训练数据）
- 标签使用效果反馈（下游系统使用标签后的效果回流，如管理后备实际晋升率）
- 规则效果追踪（命中率 / 误判率 / 人工修正率 → 规则健康度评分）
- 语义模型持续学习（企业特有口径通过使用数据持续校准）

### 可解释层 AI

- AI 决策过程可视化（AI 看到了什么、考虑了什么、为什么选这个标签）
- 反事实解释（"如果张三没有'推荐系统'经历，就不会被打上'算法方向'标签"）
- 置信度分布展示（候选标签的概率排序）
- 对比解释（"张三是核心骨干，李四不是，差异在于绩效 A vs B"）

## 第五层：领域服务层

系统核心的领域能力，被上层应用模块调用：

### 标签资产服务

- 类目 CRUD + 编码校验（CAT_ 前缀）
- 标签 CRUD + 编码校验（TAG_ 前缀）
- 标签迁移（归属变更，不触发重算）
- 标签合并（引用迁移 + 结果归并 + 源标签归档）
- 标签依赖图（标签间前置依赖 + 跨类目互斥规则）

### 规则引擎服务

- 结构化规则解析（DSL 解析器：rule / when / then / end / explain）
- 条件匹配（单值 / 枚举 / 区间 / 多分支）
- 适用范围匹配（组织 + 含下级 / 岗位序列 / 人群包；同类型 OR，跨类型 AND）
- 规则生命周期（草稿 → 发布 → 冻结 → 复制修订 → 停用）
- 冲突消解引擎（精度 > 优先级 > 发布时间）
- 标签依赖校验（打标前检查前置标签是否存在）

### 执行任务服务

- 模拟打标执行（不写入正式结果）
- 正式打标执行（含重算，写入正式结果）
- 任务状态管理（待运行 → 运行中 → 运行成功/失败 → 待提交 → 已入库）
- 结果对比生成（当前标签 vs 本次标签）
- 证据链生成（来源规则 / 命中字段 / 原始值 / 条件快照 / 适用范围快照 / 解释摘要）
- 事件驱动增量重算（晋升 / 入职 / 绩效 / 组织调整 → 触发重算）
- 定时批量重算调度

### 数据血缘服务

- 正向解释（为什么打上这个标签）
- 反向解释（为什么没有打上某个标签）
- 规则冲突可视化
- 标签变更历史时间线
- 标签溯源一键查看

### 权限与审计服务

- 角色管理（TAG_ADMIN / RULE_ADMIN / RULE_PUBLISHER / RESULT_VIEWER / SYS_ADMIN）
- 对象级权限控制
- 操作日志记录（CREATE / UPDATE / PUBLISH / STOP / DELETE / RECALC）
- 标签可见性控制（敏感标签仅特定角色可见）
- 数据权限与标签权限联动

### 多租户服务

- BU 级标签体系隔离
- 集团统一标签 + BU 自定义标签分层
- 租户级数据隔离

- tag_dependency（标签依赖关系）

- ai_training_data（AI 训练数据）
- event_trigger_config（事件触发配置）

### 缓存与临时存储

- 模拟打标临时结果（sessionStorage / 临时表）
- 规划智能体草稿（localStorage）
- AI 对话上下文（会话级）
