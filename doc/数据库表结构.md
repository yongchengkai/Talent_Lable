# 人才打标系统数据库表结构

> 基于 Flyway 迁移脚本 V1 ~ V8 整理，反映当前最新数据库设计。

## 1. 表清单总览

共 20 张表，分为 5 组：

| 分组 | 表名 | 用途 |
|------|------|------|
| 员工数据 | employee | 员工基础信息 |
| 标签体系 | tag_category | 标签类目 |
| | tag_definition | 标签定义 |
| 规则引擎 | tag_rule | 规则主表 |
| | tag_rule_branch | 规则分支 |
| | tag_rule_output | 规则输出标签 |
| | tag_rule_condition | 规则条件 |
| | tag_rule_scope | 规则适用范围 |
| | ai_rule_config | AI 语义规则配置 |
| 执行与结果 | calc_task | 打标任务 |
| | calc_task_rule | 任务-规则关联 |
| | employee_tag_result | 标签结果 |
| | employee_tag_result_detail | 结果证据明细 |
| 变更感知 | employee_change_log | 员工变更日志 |
| | change_notification | 变更通知 |
| AI 能力 | ai_skill | AI 技能定义 |
| | ai_skill_log | 技能调用日志 |
| | ai_pending_operation | 待确认操作 |
| | ai_model_config | AI 模型配置 |
| | ai_chat_session | 对话会话 |
| | ai_chat_message | 对话消息 |

## 2. 表结构详情

### 2.1 employee

员工基础信息，是规则判断的数据源。由外部系统写入，PG 触发器自动记录字段级变更。

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGSERIAL | 是 | 主键 |
| employee_no | VARCHAR(64) | 是 | 员工编号，唯一 |
| name | VARCHAR(128) | 是 | 姓名 |
| org_id | BIGINT | 是 | 所属组织 ID |
| org_name | VARCHAR(128) | 是 | 组织名称 |
| org_path | VARCHAR(1024) | 是 | 组织路径，用于判断是否属于某组织及下级 |
| position_sequence_code | VARCHAR(64) | 否 | 岗位序列编码 |
| position_sequence_name | VARCHAR(128) | 否 | 岗位序列名称 |
| job_family_code | VARCHAR(64) | 否 | 职族编码 |
| job_family_name | VARCHAR(128) | 否 | 职族名称 |
| job_title | VARCHAR(128) | 否 | 职务 |
| grade_level | VARCHAR(64) | 否 | 职级，如 P7 |
| birth_date | DATE | 否 | 出生日期 |
| hire_date | DATE | 否 | 入职日期 |
| education | VARCHAR(64) | 否 | 学历 |
| university | VARCHAR(256) | 否 | 毕业院校 |
| resume_text | TEXT | 否 | 简历全文 |
| project_experience | TEXT | 否 | 项目经历 |
| employment_type | VARCHAR(64) | 否 | 用工类型 |
| employee_status | VARCHAR(32) | 是 | 员工状态（在职/离职等） |

| created_at | TIMESTAMP | 是 | 创建时间 |
| updated_at | TIMESTAMP | 是 | 更新时间 |

## 3. 触发器

### fn_employee_change_log()

- 触发条件：AFTER INSERT OR UPDATE ON employee
- INSERT：所有非空字段写入 employee_change_log
- UPDATE：用 `IS DISTINCT FROM` 比对，只记录变化的字段
- 定时任务每 60 秒扫描 `processed = FALSE` 的记录，匹配受影响规则后生成 change_notification
