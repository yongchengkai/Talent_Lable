标签体系对象状态与操作说明

覆盖对象：标签类目、标签、条件打标规则、智能打标规则、模拟打标任务、正式打标任务。依据当前后端 Service 约束与前端页面交互逻辑整理（截至本仓库当前代码）。

标签类目 tag_category

状态 可用操作 操作限制 说明 / 依据
启用
  • 编辑
  • 停用（满足条件时）
  • 删除（满足条件时）
  • 直接停用（若仍有启用标签）
  • 直接删除（若仍有任意标签）
  • 停用前提：activeTagCount = 0
  • 删除前提：tagCount = 0
 类目用于承载标签；需先清空类目影响再做停用/删除。
停用
  • 启用
  • 编辑
  • 删除（满足条件时）
  • 删除（若仍有标签）
  • 删除前提：类目下无标签（tagCount = 0）
 停用不影响历史结果，仅限制该类目继续承载新标签治理动作。

标签 tag_definition

状态 可用操作 操作限制 说明 / 依据
启用
  • 编辑
  • 迁移类目
  • 停用（满足条件时）
  • 删除（满足条件时）
  • 停用限制：若被任意规则引用则不可停用
  • 删除限制：若被任意规则引用则不可删除
  • 删除限制：若存在历史打标结果/证据引用则不可删除
 删除状态已放宽：启用/停用均可删，但必须通过“规则引用 + 历史引用”校验。
停用
  • 启用
  • 编辑
  • 迁移类目
  • 删除（满足条件时）
  • 删除限制：若被任意规则引用则不可删除
  • 删除限制：若存在历史打标结果/证据引用则不可删除
 删除是否可用不看标签启停状态，只看引用校验结果。

条件打标规则 tag_rule(rule_type=STRUCTURED)

状态 可用操作 操作限制 说明 / 依据
未发布
  • 编辑
  • 发布
  • 复制
  • 删除
  • 撤销发布
  • 删除限制：若被任意任务引用（模拟任务 SIMULATION / 正式任务 FORMAL）则不可删除
  • 规则编码约束：必须以 CR_ 开头
 新增规则默认未发布；未发布状态可编辑。删除是否可用不看发布状态，只看“是否被任务引用”。
已发布
  • 编辑（满足条件时）
  • 复制（生成新草稿）
  • 删除（满足条件时）
  • 撤销发布（满足条件时）
  • 查看关联正式任务
  • 编辑限制：若已在任一正式任务中运行成功，且当前不在正式任务运行中，则不可编辑
  • 删除限制：若被任意任务引用（模拟任务 SIMULATION / 正式任务 FORMAL）则不可删除
  • 再次发布
  • 撤销发布限制：若被正式任务（FORMAL）引用则不可撤销
 编辑口径调整为：未发布可自由编辑；已发布规则只要未在任何正式任务中运行成功，或处于正式任务运行过程中，均可编辑。删除口径调整为：不再要求“必须未发布”，改为“未被任何任务引用（SIMULATION/FORMAL）即可删除”；撤销后会失效该规则当前有效标签结果（valid_flag=false）。

智能打标规则 tag_rule(rule_type=AI_SEMANTIC)

状态 可用操作 操作限制 说明 / 依据
未发布
  • 编辑
  • 发布
  • 复制
  • 删除
  • 撤销发布
  • 删除限制：若被任意任务引用（模拟任务 SIMULATION / 正式任务 FORMAL）则不可删除
  • 规则编码约束：必须以 AR_ 开头
 新增规则默认未发布；未发布状态可编辑。删除是否可用不看发布状态，只看“是否被任务引用”。
已发布
  • 编辑（满足条件时）
  • 复制（生成新草稿）
  • 删除（满足条件时）
  • 撤销发布（满足条件时）
  • 查看关联正式任务
  • 编辑限制：若已在任一正式任务中运行成功，且当前不在正式任务运行中，则不可编辑
  • 删除限制：若被任意任务引用（模拟任务 SIMULATION / 正式任务 FORMAL）则不可删除
  • 再次发布
  • 撤销发布限制：若被正式任务引用则不可撤销
 与条件规则共用同一套判定口径：未发布可自由编辑；已发布规则只要未在任何正式任务中运行成功，或处于正式任务运行过程中，均可编辑。删除不再按发布状态限制，改为“未被任何任务引用（SIMULATION/FORMAL）即可删除”。

模拟打标任务 calc_task(task_mode=SIMULATION)

运行状态 可用操作 操作限制 说明 / 依据
待运行（INIT）
  • 编辑
  • 运行
  • 删除
  • 复制
  • 查看结果（通常无结果）
 模拟任务用于预演验证，不进入正式审批生效链路。
运行中
  • 查看详情/等待完成
  • 编辑
  • 删除
  • 再次运行
  • 运行中硬锁定（后端拒绝编辑/删除）
 后端依据：CalcTaskServiceImpl.update/delete。
运行成功
  • 查看结果
  • 撤销（清理结果并回待运行 INIT）
  • 重新运行
  • 直接编辑（需先撤销）
  • 直接删除（需先撤销）
  • 删除流程限制：需先撤销后再删除（前后端一致）
 运行成功后，编辑/删除均需先撤销，前后端规则已对齐。
运行失败
  • 编辑
  • 重试运行
  • 删除
  • 提交审批
 模拟任务不走提交审批链路。

正式打标任务 calc_task(task_mode=FORMAL)

状态组合 可用操作 操作限制 说明 / 依据
待运行 + 待提交（INIT + PENDING）
  • 编辑
  • 运行
  • 删除
  • 提交审批
 正式任务初始阶段。
运行中 + 待提交
  • 查看详情
  • 编辑
  • 删除
  • 再次运行
  • 运行中硬锁定（后端拒绝编辑/删除）
 后端依据：CalcTaskServiceImpl.update/delete/run。
运行成功 + 待提交
  • 提交审批
  • 查看结果
  • 重新运行
  • 撤销
  • 编辑
  • 提交限制：仅正式任务且运行成功时可提交
 提交后将进入审批流程。
运行成功 + 已提交
  • 审批通过 / 驳回（审批页）
  • 查看结果
  • 编辑
  • 重新运行
  • 撤销
  • 流程限制：仅审批动作可推进状态
 审批入口：审批页通过/驳回。
运行成功 + 已审批
  • 查看结果（已生效）
  • 重新运行（当前实现允许）
  • 撤销（当前实现允许）
  • 编辑
 审批通过会激活结果并替换历史同员工同标签结果。
运行成功 + 已驳回
  • 重新运行
  • 撤销
  • 查看结果
  • 编辑
 可重跑后再次提交审批。
运行失败 + 待提交
  • 编辑
  • 重试运行
  • 删除
  • 提交审批
 失败任务需修复后重跑。

审批 审批动作约束

审批动作约束：approve/reject 仅允许提交状态为“已提交（SUBMITTED）”。
